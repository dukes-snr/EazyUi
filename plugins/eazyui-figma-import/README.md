# EazyUI Figma Import

Local Figma plugin for importing the `eazyui.figma-scene` payload generated by EazyUI.

## Files

- `manifest.json`
- `code.js`
- `ui.html`

## Install In Figma

1. Open Figma desktop.
2. Go to `Plugins` -> `Development` -> `Import plugin from manifest...`
3. Select:

```text
plugins/eazyui-figma-import/manifest.json
```

## Import Flow

1. In EazyUI click `Copy Figma Payload`.
2. In Figma run `EazyUI Figma Import`.
3. Click `Paste From Clipboard`.
4. Click `Import Into Figma`.

## What The Plugin Creates

- One Figma frame per exported screen.
- Nested frames for HTML containers.
- Native text nodes for text payload nodes.
- Rectangle nodes with image fills for image payload nodes when Figma can resolve the image source.
- Optional local variables, styles, and repeated component masters from payload design-system metadata.

## Import Config

The plugin config modal now includes toggles for:

- `Create variables`
- `Create styles`
- `Create components`

## Current Limits

- The payload is EazyUI-owned, not Figma's private clipboard format.
- CSS fidelity is first-pass, not browser-complete.
- Auto-layout is inferred from HTML/CSS and still depends on safe flex/container heuristics.
- Some CSS filters and browser-only blend/compositing behaviors still do not have exact native Figma equivalents.
- Remote images can still fail if the source is blocked or unsupported by Figma image import.
